# 文档摘要

- 本交付不是直接照搬某一个开源仓库，而是把 `TrendLineAutomation` 的回归通道包络思路与 `pytrendline` 的触碰计数、突破过滤思路混合后，改写成适合 `439,488` 根 `ETHUSDT 5m` K 线批量运行的版本。
- 参考实现版本为 `TrendLineAutomation@63b1429` 与 `pytrendline@d575f95`；当前实现明确放弃 `pytrendline` 的 `O(N^3)` 穷举趋势线搜索，改用滚动 `48` 根窗口回归通道，以控制运行成本。
- 检测逻辑只把“真正像趋势线”的窗口留下来：要求支撑线和阻力线都至少有 `3` 次分离触碰，通道高度介于 `1.2` 到 `12 ATR`，然后再判断是否发生有效上破/下破。
- 图表不只画框和信号点，还把识别时最重要的感知点简单连起来：支撑触点 `S1/S2/S3`、阻力触点 `R1/R2/R3`、支撑/阻力边界线、以及最后一次关键触碰到突破点的虚线。
- 全样本基线共得到 `41` 笔交易，其中 `18` 笔做多、`23` 笔做空；生成案例图时命中不足 `100` 张，因此实际输出全部 `41` 张。
- 基线回测净值略亏，说明“趋势线被多次测试后突破”的结构语义可以稳定识别出来，但当前入场和风控组合还不足以形成强正期望。

# 量化定义

## 模式 1：上破阻力趋势线

| 字段 | 内容 |
| --- | --- |
| 所需字段 | `timestamp/open/high/low/close/volume` |
| 衍生特征 | `atr`、`close_pos`、`body_ratio`、`trend_window_start_idx/end_idx`、`trend_slope`、`trend_channel_type`、`support_line_*`、`resistance_line_*`、`channel_height_atr`、`support_touch_*`、`resistance_touch_*` |
| 关键感知点 | 最近 `48` 根窗口里的 `S1/S2/S3`、`R1/R2/R3`、支撑边界、阻力边界、突破点 |
| 简单连线 | 图上默认连支撑触点折线、阻力触点折线、完整支撑/阻力边界线，并用短虚线把最后一个关键阻力触点连到上破触发点 |
| 判定逻辑 | 对最近 `48` 根收盘做最小二乘回归，得到中轴；再用窗口内高低点求支撑/阻力偏移，形成通道边界。若支撑和阻力都至少有 `3` 个分离触碰，且通道高度在 `1.2-12 ATR` 之间，则认为通道有效。随后要求前一根收盘仍未明显站上阻力线、本根收盘有效上破阻力线 `0.2 * ATR`，同时 `close_pos >= 0.60`、`body_ratio >= 0.45`。 |
| 确认条件 | `trend_channel_valid == True` 且 `bullish_breakout_signal == True`。 |
| 失效条件 | 入场后若重新回到止损位，或未来 `24` 根内最低价重新跌回当前阻力线之下，则分别记作交易止损和研究标签上的失败突破。 |
| 待验证假设 | `line_window=48`、`touch_tolerance_atr=0.18`、`breakout_tolerance_atr=0.20`、`close_pos/body_ratio` 阈值都属于经验假设，需要按品种与周期重新校准。 |

| 判定逻辑 | 在同一套回归通道基础上，要求前一根收盘仍未明显跌破支撑线、本根收盘有效下破支撑线 `0.2 * ATR`，同时 `close_pos <= 0.40`、`body_ratio >= 0.45`。 |

| 待验证假设 | 当前实现要求支撑线与阻力线同时有效，这会明显压缩样本量，但有助于减少“只是一侧碰线”的伪趋势线。 |

# 交易语义

## 上破阻力趋势线

- 入场触发：突破 K 收盘即做多。
- 止损位置：当前支撑线下方减去 `0.12 * ATR`，且最小风险不低于 `0.7 * ATR`。
- 目标位/测量目标：`max(0.9 * 通道高度, 1.8R)`。
- 失败条件：重新跌回止损位，或事后标签上跌回阻力线下方。
- 适用品种或周期假设：当前只验证 `ETHUSDT 5m`；更平滑的慢周期可能需要更长窗口。
- 建议统计指标：上破命中数、上破后 `24` 根内测量目标命中率、失败突破率、突破后最大顺势/逆势波动。

## 下破支撑趋势线

- 入场触发：跌破 K 收盘即做空。
- 止损位置：当前阻力线上方加上 `0.12 * ATR`，且最小风险不低于 `0.7 * ATR`。
- 目标位/测量目标：`max(0.9 * 通道高度, 1.8R)`。
- 失败条件：重新上破止损位，或事后标签上重新站回支撑线上方。
- 适用品种或周期假设：更适合波动连续、趋势边界清晰的市场；噪声盘整里会显著增加假突破。
- 建议统计指标：下破命中数、测量目标命中率、失败跌破率、跌破后持仓 `MFE/MAE`。

# 交易策略

| 项目 | 规则 |
| --- | --- |
| 方向 | 上破阻力线做多，下破支撑线做空 |
| 入场规则 | 信号 K 收盘即入场，不等待二次回踩 |
| 出场规则 | 命中止损、命中目标位，或持仓满 `48` 根 K 线离场 |
| 止损规则 | 多头用当前支撑线下方 `0.12 * ATR`，空头用当前阻力线上方 `0.12 * ATR`，且最小风险 `0.7 * ATR` |
| 止盈/目标位规则 | `max(0.9 * 通道高度, 1.8R)` |
| 过滤条件 | 通道窗口 `48` 根；支撑/阻力都至少 `3` 次触碰；通道高度 `1.2-12 ATR`；突破 K 需满足收盘位置与实体比例约束 |
| 风险管理假设 | 固定名义仓位 `10000 USDT`，每边 `2 bps` 含手续费与滑点，不做复利 |
| 建议回测指标 | 交易笔数、胜率、平均盈亏、总盈亏、平均 `R`、总 `R`、最大回撤、Profit Factor、净值、时间离场占比 |

# 验证与风险

- `label_trendline_outcomes()` 使用未来 `24` 根 K 线，只能用于研究标签，不能用于实时交易。
- 当前 `48` 根窗口与 `3` 次触碰阈值是经验设定；窗口变长会提升结构稳定性但减少机会，窗口变短则会增加噪声。
- 当前实现要求支撑线和阻力线同时有效，这能减少伪信号，但也可能漏掉“单边明显、另一边较弱”的真实趋势线。
- 本次只验证 `ETHUSDT 5m`，没有做年份切分、滚动样本外或多品种迁移验证；下一步应至少做年份分层和前向验证。
- 若继续调参，最值得优先验证的是 `line_window`、`min_touches`、`breakout_tolerance_atr` 和入场方式；尤其是改成“突破后的回踩确认”时，要防止只在当前样本上过拟合。
