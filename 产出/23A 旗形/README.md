# 23A 旗形

来源文档：`阿布课程语音转文字/23A 旗形.txt`  
处理方式：`$abu-pattern-quant`  
默认样本数据：`data/binance_um_perp/ETHUSDT/5m/ETHUSDT-5m-history.parquet`

## 文档摘要

- 这篇文档的核心不是普通旗形延续，而是“最终旗形”：趋势中的最后一个旗形会在突破后失败，并进入反转或交易区间。
- 最终旗形本质上是趋势后期的交易区间，尤其是持续 `20` 根或更多 K 线、接近支撑阻力或等距目标时，突破失败概率上升。
- 最终多头旗形常见于多头趋势后期的横盘、三角形、楔形顶、阻力位下整理；最终空头旗形常见于空头趋势后期的反弹整理、楔形空头旗形、支撑位上方停滞。
- 文档反复强调两个判别核心：`趋势已运行较久` 与 `旗形已扩展成交易区间`。这两个条件叠加时，交易者应从顺势逻辑切换到警惕失败突破和反转逻辑。
- 原文还强调 `80%` 交易区间突破失败这一经验法则，因此最终旗形的量化必须把“失败突破”与“回到共识区域”单独建模。

## 形态归纳

1. 常规旗形
- `原文线索`：趋势中的回调、小三角形、I-I-I、双顶/双底旗形。
- `归一化模式名`：常规旗形。
- `为什么归到这里`：原文把旗形定义为趋势中的回调，默认先看趋势延续。

2. 最终多头旗形
- `原文线索`：多头趋势后期、`20` 根或更多 K 线、阻力位下横盘、等距目标附近、楔形更高高点、买入高潮。
- `归一化模式名`：最终多头旗形。
- `为什么归到这里`：这些条件共同指向“向上突破可能失败并转为空头波段或交易区间”。

3. 最终空头旗形
- `原文线索`：空头趋势后期、支撑位上方整理、低2、楔形空头旗形、抛售高潮后停滞、向下突破失败。
- `归一化模式名`：最终空头旗形。
- `为什么归到这里`：这些条件共同指向“向下延续可能是最后一段，随后反转向上”。

## 量化定义

1. 常规旗形
- `所需字段`：`timestamp/open/high/low/close/volume`
- `衍生特征`：`atr20`、`ema20`、`ema50`、`rolling_high_flag`、`rolling_low_flag`、`close_pos`、`flag_range`
- `判定逻辑`：先识别先行趋势，再寻找 `4-10` 根窄幅整理，且 `flag_range <= 3 * atr20`，整理方向逆趋势或接近水平。
- `确认条件`：多头旗形要求收盘上破前一根旗形上沿且 `close_pos >= 0.7`；空头旗形要求收盘下破前一根旗形下沿且 `close_pos <= 0.3`。
- `失效条件`：突破后 `2-6` 根内重新回到旗形区间，或快速穿越旗形另一侧边界。
- `待验证假设`：`4-10` 根、`3 * atr20`、`0.7/0.3` 作为 ETHUSDT 5m 的起始阈值，后续需要滚动校准。

2. 最终多头旗形
- `所需字段`：常规旗形字段，加 `rolling_high_late`、`rolling_low_late`、`rolling_high_60`
- `衍生特征`：`uptrend_score`、`late_flag_range`、`near_resistance`、`buy_climax`
- `判定逻辑`：过去 `20` 根中趋势向上占优；当前整理持续 `10-20` 根并扩展成交易区间；价格靠近最近 `60` 根高点或等距目标；随后发生向上突破。
- `确认条件`：向上突破后 `6` 根内回到旗形内部，或跌破旗形下沿。
- `失效条件`：突破后继续有效创高，且 `6` 根内未回到区间。
- `待验证假设`：`uptrend_score >= 14`、阻力接近度 `0.5 * atr20`、失败窗口 `6` 根。

3. 最终空头旗形
- `所需字段`：常规旗形字段，加 `rolling_high_late`、`rolling_low_late`、`rolling_low_60`
- `衍生特征`：`downtrend_score`、`late_flag_range`、`near_support`、`sell_climax`
- `判定逻辑`：过去 `20` 根中趋势向下占优；当前出现 `10-20` 根反弹整理、楔形反弹或低2；价格接近最近 `60` 根低点或等距目标；随后发生向下突破。
- `确认条件`：向下突破后 `6` 根内回到区间，或上破旗形上沿。
- `失效条件`：突破后继续有效创新低，且 `6` 根内未回到区间。
- `待验证假设`：支撑接近度、抛售高潮阈值、失败突破确认窗口都需继续回测。

## 交易语义

1. 常规旗形
- `入场触发`：顺趋势突破旗形边界并获得强收盘确认。
- `止损位置`：旗形另一侧边界外 `0.3-0.5 * atr20`。
- `目标位/测量目标`：前一冲动腿等距投影，或先看第二波延续。
- `失败条件`：突破后快速回区间并反向穿越另一侧边界。
- `适用品种或周期假设`：优先用于 `ETHUSDT 5m`，其他品种需重估波动阈值。
- `建议统计指标`：突破后 `6/12` 根收益、回撤、目标命中率、失败率。

2. 最终多头旗形
- `入场触发`：向上突破失败后，跌破旗形下沿或更低高点信号 K 低点。
- `止损位置`：失败突破高点上方 `0.5 * atr20`。
- `目标位/测量目标`：旗形中轴、起涨点、最近交易区间下沿。
- `失败条件`：再次站回旗形上沿并创有效新高。
- `适用品种或周期假设`：趋势后期、阻力附近、买入高潮后更有效。
- `建议统计指标`：失败突破后 `10` 根收益、最大顺势波幅、突破到失败的 bar 数。

3. 最终空头旗形
- `入场触发`：向下突破失败后，上破旗形上沿或更高低点/微双底信号 K 高点。
- `止损位置`：失败突破低点下方 `0.5 * atr20`。
- `目标位/测量目标`：旗形中轴、最近反弹高点、空头冲动起点附近。
- `失败条件`：再次跌回旗形下沿并创有效新低。
- `适用品种或周期假设`：空头趋势后期、支撑附近、抛售高潮后更有效。
- `建议统计指标`：失败突破后 `10` 根收益、反转长度、二次入场成功率。

## 交易策略

1. 顺势旗形突破策略
- `方向`：顺先行趋势。
- `入场规则`：`bull_flag_breakout` 或 `bear_flag_breakout` 后下一根 K 线开盘入场。
- `出场规则`：触及 `2R` 目标、止损、或 `12` 根内无延续则时间离场。
- `止损规则`：旗形另一侧边界外加 `0.5 * atr` 缓冲。
- `止盈/目标位规则`：固定 `2R` 目标。
- `过滤条件`：过宽旗形、风险为负、或下一根不存在时跳过。
- `风险管理假设`：单笔按 `1R` 归一化；同向多笔信号独立统计。
- `建议回测指标`：胜率、平均 `R`、总 `R`、持仓 bar 数。

2. 最终旗形失败突破反转研究策略
- `方向`：逆原趋势。
- `入场规则`：对 `final_bull_flag_confirmed`，在失败确认后的下一根开盘做空；对 `final_bear_flag_confirmed`，在失败确认后的下一根开盘做多。
- `出场规则`：触及 `2R` 目标、止损、或 `16` 根内无延续则时间离场。
- `止损规则`：突破极值或旗形边界外加 `0.5 * atr` 缓冲。
- `止盈/目标位规则`：固定 `2R` 目标。
- `过滤条件`：只用于研究标签，不作为实时信号；需要失败突破已经被确认。
- `风险管理假设`：这是事后研究策略，目的是验证文档中的“最终旗形”是否值得单独建模。
- `建议回测指标`：命中数量、平均 `R`、样本稀疏程度、胜率。

## 验证与风险

- `FLAG_WINDOW`、`LATE_FLAG_WINDOW`、`14/20` 趋势分数、`0.5 * atr` 的支撑阻力容忍度仍是待验证假设。
- “`20` 根或更多 K 线”“买入/抛售高潮”“`80%` 交易区间突破失败”来自课程经验法则，不应直接视为稳定参数。
- `future_low_n`、`future_high_n`、`bull_breakout_failed`、`bear_breakout_failed`、`final_*_confirmed` 使用了未来 K 线，只能用于研究标签，不能伪装成实时信号。
- 最终旗形样本极少，当前 `ETHUSDT 5m` 全样本中只有 `3` 个最终多头旗形确认、`0` 个最终空头旗形确认，说明当前参数偏保守且样本稀疏。
- 更稳妥的下一步是：
  - 放宽或重估失败突破阈值
  - 单独标注支撑阻力与等距目标
  - 做滚动前向验证，而不是只看全样本总 `R`
