# 文档摘要

- 原文中的“最终旗形”本质是晚期趋势中的最后一个回调、三角形或水平交易区间，表面上像顺势旗形，实际上更容易演化为失败突破后的反转。
- 核心上下文不是单独一个旗形，而是“趋势已经走了 20 根以上 K 线、回调很少、靠近支撑阻力或测量目标、区间开始横向化”。
- 水平交易区间被当作共识区处理，任何向上或向下脱离共识区的突破，都要优先评估是否会快速回到区间内部。
- 多头最终旗形对应“晚期多头趋势 + 水平/收敛旗形 + 上破失败”；空头最终旗形对应“晚期空头趋势 + 水平/收敛旗形 + 下破失败”。
- 原文反复强调三角形、楔形、I-I-I、双顶/双底、阻力/支撑测试与高潮衰竭，这些都在本实现里被压缩为结构过滤器，而不是单独拆成独立策略。

# 量化定义

## 模式 1：最终多头旗形

| 字段 | 内容 |
| --- | --- |
| 所需字段 | `timestamp/open/high/low/close/volume` |
| 衍生特征 | `atr`、`ema20`、`ema50`、`uptrend_score`、`buy_climax_count`、`overlap_ratio`、`late_flag_height_atr`、`late_flag_drift_atr`、`near_resistance`、`flag_high_pre`、`flag_low_pre` |
| 判定逻辑 | 最近 20 根内 `uptrend_score >= 14`；最近 20 根形成宽度不超过 `4.4 * ATR` 的横向/收敛区间；区间漂移不超过 `1.1 * ATR`；区间重叠率至少 `0.55`；价格位于 `ema50` 上方；且满足 `near_resistance`、`buy_climax_extension`、`buy_climax_count >= 5` 三者之一。 |
| 确认条件 | 在上述上下文中，12 根旗形窗口的前高被向上突破，且突破 K 线 `close_pos >= 0.62`、`body_ratio >= 0.45`；随后 6 根内重新跌回旗形上沿下方并给出偏空确认 K。 |
| 失效条件 | 失败突破后的做空确认未出现；或出现后价格重新突破失败高点并触发止损。 |
| 待验证假设 | `20` 根趋势长度、`4.4 * ATR` 区间宽度、`0.55` 重叠率、`0.62` 收盘位置阈值都来自课程语义的工程化翻译，未做跨品种校准。 |

# 交易语义

## 最终多头旗形

- 入场触发：晚期多头趋势中的上破，在 6 根 K 线内回落回旗形上沿下方，并出现偏空确认 K 后做空。
- 止损位置：失败突破段最高点上方再加 `0.15 * ATR`，且最小止损距离不低于 `0.7 * ATR`。
- 目标位/测量目标：取 `max(旗形高度, 1.5R)`，等价于先看回归共识区，再看至少一段旗形高度的反向延伸。
- 失败条件：确认后重新站回失败突破高点上方。
- 适用品种或周期假设：当前验证样本是 `ETHUSDT 5m`，更高周期或波动更低的市场需要重校准。
- 建议统计指标：失败突破转化率、入场后 12/24/36 根的 MFE/MAE、扣费后胜率、扣费后 profit factor。

## 最终空头旗形

- 入场触发：晚期空头趋势中的下破，在 6 根 K 线内回到旗形下沿上方，并出现偏多确认 K 后做多。
- 止损位置：失败跌破段最低点下方减去 `0.15 * ATR`，且最小止损距离不低于 `0.7 * ATR`。
- 目标位/测量目标：取 `max(旗形高度, 1.5R)`，表达的是“回到共识区后再做一段反抽”。
- 失败条件：确认后重新跌破失败低点。
- 适用品种或周期假设：当前仅对 `ETHUSDT 5m` 回测；强趋势日内品种较适配，低波动区间品种需重估。
- 建议统计指标：下破失败后的反抽幅度、入场后 12/24/36 根持仓分布、净值回撤、交易成本敏感性。

# 交易策略

| 项目 | 规则 |
| --- | --- |
| 方向 | 上破失败做空，下破失败做多 |
| 入场规则 | 突破后最多等待 6 根 K 线；重回旗形内部并出现反向确认 K 时，以确认 K 收盘价入场 |
| 出场规则 | 命中止损、命中目标位，或持仓满 36 根 K 线平仓 |
| 止损规则 | 失败突破段极值外加/减 `0.15 * ATR`，且最小止损为 `0.7 * ATR` |
| 止盈/目标位规则 | `max(旗形高度, 1.5R)` |
| 过滤条件 | 晚期趋势、横向/收敛旗形、接近支撑阻力或高潮衰竭、突破 K 收盘位置强 |
| 风险管理假设 | 固定名义仓位 `10000 USDT`，每边 `2 bps` 计入手续费与滑点，不做复利 |
| 建议回测指标 | 交易笔数、胜率、平均盈亏、总盈亏、平均 `R`、总 `R`、最大回撤、profit factor |

# 验证与风险

- `flag_window=12`、`late_flag_window=20`、`trend_score_min=14`、`range_width_atr=3.2` 等都是根据课程语言压缩出来的研究阈值，不应直接外推到别的市场。
- `label_final_flag_outcomes()` 使用了未来 8 根 K 线，只能用于事后统计，不能当作实时信号。
- 当前样本只覆盖 `ETHUSDT 5m`，没有做训练/验证/前向拆分；下一步应按年份或滚动窗口做分段验证。
- 交易成本对结果非常敏感：当前在每边 `2 bps` 下为小幅正收益，但边际很薄；若执行成本回到更高水平，绩效很容易再次转弱。
- 当前策略把所有最终旗形统一成一套出场规则，没有单独区分“仅回到共识区”和“发展成完整反转”的两类场景，可能压低了收益质量。
- 若后续放宽参数或引入筛选器，必须做样本外验证，避免只是在当前四年样本上追求更漂亮的回测曲线。
