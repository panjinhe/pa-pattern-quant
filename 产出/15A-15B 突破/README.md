# 文档摘要

- `15A` 把突破定义得非常宽，几乎每根趋势 K 线都可以视为某种突破；但量化时不能直接把“所有趋势 K 线”都当成信号，所以本实现把突破锚定到最近 `20` 根的显式交易区间高低点。
- `15B` 的核心补充是：突破是否成功，不看“突破 K 线看起来多强”，而看它能否在后面 `1-4` 根内得到同方向 follow-through；因此本次专门把“突破 K 线”和“后续确认 K 线”分开建模。
- 文本另一条主线是阿布的 `80%` 规则：交易区间里的大多数突破会失败。所以本实现没有只做追突破，而是同时输出 `follow-through 延续` 和 `失败突破反手` 两类信号。
- 图表里除了区间框和入场点，还显式连出了支撑/阻力触点、区间上下沿、突破点和确认点，便于人工巡检“为什么这里算突破”“为什么这里算失败突破回归”。
- 全样本基线共回测 `721` 笔交易，其中 `437` 笔是 follow-through 延续，`284` 笔是失败突破反手；两类都能稳定抓到样本，但净值仍显著为负。
- 分家族看，`下破失败做多` 最稳，胜率 `49.0323%`、平均净盈亏 `-2.7860 USDT`；`下破后续做空` 最弱，胜率仅 `35.7798%`、平均净盈亏 `-8.5039 USDT`。

## 模式 1：突破 + follow-through 延续

| 字段 | 内容 |
| --- | --- |
| 所需字段 | `timestamp/open/high/low/close/volume` |
| 衍生特征 | `atr`、`close_pos`、`body_ratio`、`ema20/ema50`、`range_high_pre`、`range_low_pre`、`range_mid_pre`、`range_height_atr`、`support_touch_*`、`resistance_touch_*`、`bull_breakout_bar`、`bear_breakout_bar`、`bull_followthrough_signal`、`bear_followthrough_signal` |
| 关键感知点 | 区间上沿/下沿、最近三次边界触点、突破 K 线、后续确认 K 线 |
| 简单连线 | 图上默认连 `S1/S2/S3`、`R1/R2/R3`、区间上下沿；对成功突破，再用虚线连“最后一个关键边界触点 -> 突破点 -> 确认点” |
| 判定逻辑 | 以前 `20` 根 K 线的最高点/最低点定义突破前区间，要求区间高度在 `1-7 ATR` 之间，且上下边界都至少有 `2` 次分离触碰。若当前收盘突破区间边界 `0.08 * ATR`，同时 `close_pos >= 0.65 / <= 0.35`、`body_ratio >= 0.55`，则记为突破 K 线。之后 `1-4` 根内若再次向突破方向延续，并满足更宽松的 `close_pos` 与实体要求，则记为 follow-through 信号。 |
| 确认条件 | 多头需要出现 `bull_followthrough_signal`，空头需要出现 `bear_followthrough_signal`。 |
| 失效条件 | 确认后再次快速回到原区间内部，或在交易层触发止损。 |
| 待验证假设 | `range_window=20`、`breakout_buffer_atr=0.08`、`confirm_bars=4`、`followthrough_close_pos=0.58` 都是经验阈值，需要按品种与周期校准。 |

## 模式 2：交易区间失败突破反手

| 字段 | 内容 |
| --- | --- |
| 所需字段 | `timestamp/open/high/low/close/volume` |
| 衍生特征 | `overlap_ratio`、`ema_gap_atr`、`range_context`、`bull_failed_breakout_signal`、`bear_failed_breakout_signal`、`bull_failed_breakout_idx`、`bear_failed_breakout_idx` |
| 关键感知点 | 区间上沿/下沿、边界触点、突破 K 线、回到区间内部的失败确认 K 线 |
| 简单连线 | 图上默认连区间边界触点，并用虚线连“边界触点 -> 突破点 -> 回归确认点”，明确展示失败突破路径 |
| 判定逻辑 | 只有当突破发生在 `range_context == True` 的横向区间里，才允许统计失败突破。`range_context` 需要满足高重叠 (`overlap_ratio >= 0.55`)、区间高度适中、均线间距不大。若上破后 `1-4` 根内重新跌回区间上沿下方，并出现偏空确认 K 线，则记为 `bull_failed_breakout_signal`；若下破后重新站回区间下沿上方，并出现偏多确认 K 线，则记为 `bear_failed_breakout_signal`。 |
| 确认条件 | 回到区间内部，且确认 K 线方向与反手方向一致。 |
| 失效条件 | 失败突破反手后重新去测试突破极值并触发止损。 |
| 待验证假设 | 区间失败突破是否真的接近课程里的 `80%`，当前只在 `ETHUSDT 5m` 做了样本内统计，不能直接外推。 |

# 交易语义

## follow-through 延续

- 入场触发：突破后 `1-4` 根内出现同方向确认 K 线，确认 K 收盘即入场。
- 止损位置：多头放在突破 K 线低点或原区间上沿下方；空头放在突破 K 线高点或原区间下沿上方，再加 `0.12 * ATR` 缓冲，且最小风险不低于 `0.7 * ATR`。
- 目标位/测量目标：`max(0.9 * 区间高度, 1.8R)`。
- 失败条件：价格快速重回原区间内部，或交易层止损。
- 适用品种或周期假设：更适合波动连续、突破后容易跟随的市场；在慢速震荡市场里容易被来回打脸。
- 建议统计指标：follow-through 确认率、确认后 `24` 根内测量目标命中率、重回区间失败率、突破后最大顺/逆势波动。

## 失败突破反手

- 入场触发：突破后 `1-4` 根内重新回到区间内部，并给出反向确认 K 线，确认 K 收盘即反手。
- 止损位置：放在突破极值外加 `0.12 * ATR`，且最小风险不低于 `0.7 * ATR`。
- 目标位/测量目标：先看区间中轴，策略实现中取“区间中轴”和 `1.3R` 的较近者，体现短线交易语义。
- 失败条件：再次回到突破极值附近并触发止损。
- 适用品种或周期假设：更适合明显横盘、边界被反复测试的区间行情。
- 建议统计指标：失败突破转化率、回归中轴成功率、确认后 `MFE/MAE`、扣费后 win rate 和 profit factor。

# 交易策略

| 项目 | 规则 |
| --- | --- |

# 验证与风险

- 课程里“每根趋势 K 线都是突破”是非常宽的描述；当前实现把它收敛到“相对于最近 `20` 根区间边界的突破”，这是必要简化，不是对原文的逐字复刻。
- `confirm_bars=4`、`range_window=20`、`range_overlap_min=0.55` 都是经验阈值；不同周期和不同品种上很可能需要重设。
- `label_breakout_outcomes()` 使用未来 `24` 根 K 线，只能用于研究标签，不能用于实时交易。
- 当前样本只验证 `ETHUSDT 5m`，没有做年份切分、滚动样本外或多品种迁移验证；下一步应至少做前向分段验证。
- 若继续优化，最值得优先验证的是：延续类是否改成“回踩原边界再入场”、失败突破是否叠加更强的区间过滤，以及是否只保留 `bear_failed_breakout` 这类相对更稳的家族。
