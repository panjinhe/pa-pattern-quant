# 输出契约

使用这个 skill 时，响应必须是结构化 Markdown，严格按以下顺序输出；不能改成长篇散文。

## 1. 文档摘要

- 3 到 6 条要点。
- 只保留与量化有关的规则、边界、交易语义。
- 不复述课程背景，不摘抄大段原文。

## 2. 形态归纳

- 列出识别到的模式族。
- 每个模式族都要写：`原文线索`、`归一化模式名`、`为什么归到这里`。
- 如果同一段文字可映射到多个模式族，明确写主模式和次模式。

## 3. 量化定义

每个模式至少包含以下字段：

- `所需字段`
- `衍生特征`
- `判定逻辑`
- `确认条件`
- `失效条件`
- `待验证假设`

这一节必须是可执行定义，不能只写主观语义。

## 4. 交易语义

每个模式默认输出：

- `入场触发`
- `止损位置`
- `目标位/测量目标`
- `失败条件`
- `适用品种或周期假设`
- `建议统计指标`

如果原文没有直接给交易动作，也要输出最小可检验版本。

## 5. Polars 实现

- 默认输出 Python + Polars 代码块。
- 先写数据准备和列映射，再写特征列，再写模式判定列。
- 允许用函数包装重复逻辑，但不要直接接回测引擎。
- 如果阈值未定，代码里用具名常量表达，例如 `DOUBLE_TOP_TOLERANCE_ATR`。

推荐结构：

```python
import polars as pl

def prepare_ohlcv(df: pl.DataFrame) -> pl.DataFrame:
    ...

def add_features(df: pl.DataFrame) -> pl.DataFrame:
    ...

def detect_pattern(df: pl.DataFrame) -> pl.DataFrame:
    ...
```

## 6. Plotly 绘图方案

每个模式至少说明：

- `主图类型`
- `叠加图层`
- `关键标记`
- `颜色/线型语义`
- `需要显示的 hover 字段`

默认以 candlestick 为主图，不要改成抽象说明。

## 7. 验证与风险

至少覆盖：

- 哪些阈值是待验证假设
- 哪些边界定义仍可能有歧义
- 应如何用样本做人工抽查
- 如果该模式在不同市场可能失效，如何切分样本验证

## 输出风格

- 默认中文。
- 默认 UTF-8。
- 表格和列表优先，避免大段叙述。
- 如果信息不足，不要伪造；显式写 `待验证假设`。
